# `tests/`

Todos os testes da aplicação estão nesta pasta. Foi criado também um alias `~~` para facilitar a importação dos arquivos.

## `~~/unit/`

Aqui estão os testes unitários da aplicação. Todos os testes são realizados utilizando o `Jest`, que já vem configurado no boilerplate.

## `~~/e2e/`

Aqui estão os testes end-to-end da aplicação, todos realizados utilizando o `Playwright`.

Esta pasta possui um arquivo `main.tsx` porque, quando o Vite sobe a aplicação com o `mode` `e2e`, ele utiliza este arquivo como ponto de entrada. A única coisa que ele faz é utilizar o gateway que salva os dados em memória.

## `~~/factories/`

Aqui estão as factories que são utilizadas para criar os objetos de teste que podem ser utilizados por ambos os tipos de teste.
