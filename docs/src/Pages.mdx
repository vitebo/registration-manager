# `~/pages/`

Aqui ficam as páginas da aplicação, cada página é um componente React que é renderizado no `Router`.
Uma mudança importante de comentar aqui é que diferente da estrutura inicial do projeto, todas as regras de UI
ficam no componente root da página, isso é feito para facilitar a manutenção das páginas. Apesar
de ser muito convidativo criar um componente de página com lógica e acesso aos hooks e contexts, isso pode
dificultar a manutenção, reutilização e criação de teste de componentes.

Podemos usar o próprio componente `Columns` do `Dashboard` como exemplo, onde ele conhecia quais eram os
tipos de colunas que ele deveria renderizar. Isso impede que possamos reutilizar o componente `Columns` em
outra página.

Com essas atualizações agora podemos praticamente mover todos esses componentes para a pasta `~/src/components/` e
deixar as páginas apenas com a lógica de renderização e acesso aos hooks e contexts.
