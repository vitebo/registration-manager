# `~/`

No `~/src` temos alguns arquivos e pastas importantes para o projeto. Vamos falar aqui apenas sobre os
arquivos, as pastas serão abordadas mais a frente.

## `main.development.tsx` e `main.production.tsx`

Esses arquivos são os entry points da aplicação. Eles atuam apenas como um composition root, ou seja,
eles são responsáveis por injetar as dependências necessárias para a aplicação funcionar.

O `main.development.tsx` sobe a aplicação utilizando um gateway que consome os dados da API local
(`json-server`) como especificado no desafio. Já o `main.production.tsx` sobe a aplicação utilizando um
gateway que consome os dados do localstorage. Isso para simplificar as coisas e não precisar subir a API
em produção, usando o localstorage ja conseguimos uma persistência de dados.

## `constants.ts`

Esse arquivo contém algumas constantes utilizadas em toda a aplicação, não existe nada de muito especial
por aqui.

## `style.tsx`

O que antigamente era o `index.css` agora é o `style.tsx`. Esse arquivo contém o estilo global da aplicação
utilizando o `styled-components`. Nada de mais, apenas movi o `import` da font para o arquivo `index.html` e
criei algumas regras de `outline` para melhorar a acessibilidade.

## `theme.ts`

Esse arquivo contém o tema da aplicação, ele é utilizado pelo `styled-components` para definir as
(cores e sizes, border, etc...) utilizadas na aplicação. Isso apenas para conseguir alguma consistência
na hora de estilizar os componentes. Não dei muita atenção para esse arquivo, os valores na sua maioria
são retirados do `tailwind`, a ideia era realmente apenas ter um tema para a aplicação.

## `App.tsx`

Esse é o `App.tsx` como em qualquer aplicação react. Eles apenas configura os providers da aplicação e
atua como uma espécie de `layout` da aplicação.
